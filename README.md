# Bud Lite — Nostr RAG Archive

Semantic search and AI chat over a curated set of Nostr notes. Browser-based, no backend required.

**Live:** https://auggie-lahey.github.io/bud-lite/#/chat

## Architecture

```
Browser (GitHub Pages)
  ├── HuggingFace Inference API → embed query (mxbai-embed-large-v1, 1024d)
  ├── Deno CORS Proxy → Qdrant Cloud (vector search, 1607 notes)
  └── LLM API (z.ai / OpenAI / Anthropic) → synthesize answer
```

### Why the Deno proxy?

Qdrant Cloud returns proper CORS headers on real HTTP requests, but **rejects OPTIONS preflight with 403**. Browsers always send OPTIONS before any request with custom headers (`api-key`). The proxy:

- Handles CORS preflight for the browser
- Injects the Qdrant API key server-side (never exposed to browser)
- Forwards requests to Qdrant Cloud

**Proxy URL:** `https://proxy-hw1qvwkvqjc4.auggie-lahey.deno.net`
**Source:** `cors-proxy/deno-proxy.js`
**Platform:** Deno Deploy (free, no credit card)

### Deno Proxy Setup

If redeploying or setting up fresh:

1. Go to https://dash.deno.com → New Project
2. Link to `cors-proxy/deno-proxy.js` in this repo (or paste code in playground)
3. Set environment variables:
   - `QDRANT_URL` = `https://<cluster-id>.us-east-1-1.aws.cloud.qdrant.io:6333`
   - `QDRANT_API_KEY` = *(read-only API key from Qdrant Cloud)*
4. Deploy → update `CORS_PROXY` constant in `consumer-site/src/api/rag.js`

## Project Structure

```
cors-proxy/          # Deno Deploy CORS proxy for Qdrant
consumer-site/       # Vite vanilla-JS frontend (GitHub Pages)
backend/             # Python sync worker (CI only — fetches Nostr → Qdrant)
iac/                 # Terraform for Qdrant Cloud cluster
.github/workflows/   # CI: sync (daily) + deploy (on push)
```

## Data Pipeline

1. **Terraform** (`iac/`) creates Qdrant Cloud cluster + API keys
2. **Sync** (`backend/`, GitHub Actions daily at 03:17 UTC) fetches Nostr events → embeds with sentence-transformers → upserts to Qdrant
3. **Soul hints** (`consumer-site/public/soul-hints.json`) generated by LLM during sync
4. **Deploy** (GitHub Actions on push to main) builds consumer-site → GitHub Pages

## Consumer Site

Vanilla JS + Vite. No React, no framework.

### Key files
- `src/api/rag.js` — API client (Qdrant, HuggingFace, LLM)
- `src/components/rag-search.js` — Chat UI with sessions, filters, markdown
- `src/core/settings.js` — User settings (localStorage)

### Config baked at deploy time
`consumer-site/public/qdrant-config.json` — written by CI with:
- `url` — Qdrant cluster URL
- `readOnlyKey` — read-only API key
- `_h` — HuggingFace API key (reversed to bypass GitHub secret scanning)

### Local dev

```bash
cd consumer-site
npm install
npm run dev     # http://localhost:5173
```

## Technology

- **Frontend:** Vite, vanilla JS, Tailwind CSS
- **Vector DB:** Qdrant Cloud (free tier)
- **Embeddings:** HuggingFace Inference API (mxbai-embed-large-v1)
- **LLM:** Any OpenAI-compatible API (z.ai, OpenAI, Anthropic)
- **CORS Proxy:** Deno Deploy
- **CI/CD:** GitHub Actions + GitHub Pages
- **Infrastructure:** Terraform (Qdrant Cloud)
- **Sync:** Python (nostr-sdk, sentence-transformers, qdrant-client)
